# 玩法系统设计工具 — 原子开发计划

> 优先级以开发顺序排列，每条任务为可独立交付的最小单元。

---

## Phase 1：基础选项框架

- [x] 1.1 定义预设玩法系统选项的数据结构（id / name / category / mapping_desc）
- [x] 1.2 实现选项列表的渲染组件（多选 + 状态管理）
- [x] 1.3 实现选项的"映射描述"字段展示（选中后展开，定义层）
- [x] 1.4 实现选项的勾选/取消联动状态更新

---

## Phase 2：自定义系统扩展入口

- [x] 2.1 在选项列表末尾添加"自定义系统"输入入口（手动添加）
- [x] 2.2 手动添加的自定义系统自动补全 id / name 字段，category 标记为 custom
- [x] 2.3 自定义系统与预设系统统一进入同一数据流，后续流程不区分来源
- [x] 2.4 支持删除已添加的自定义系统条目

---

## Phase 3：AI 访谈兜底扩展

- [x] 3.1 设计 AI 访谈的问题集（聚焦"以上选项未覆盖的内容"）
- [x] 3.2 实现访谈对话组件（轮次问答 + 用户回答录入）
- [x] 3.3 将访谈结果解析为自定义系统条目并写入数据流
- [x] 3.4 访谈入口放置于选项选择完成后的兜底节点，不阻断主流程

---

## Phase 4：设计占比模块

- [x] 4.1 定义占比字段数据结构（system_id / weight / weight_type）
- [x] 4.2 在每个已选系统条目旁添加占比输入控件（数值输入，支持百分比）
- [x] 4.3 实现总占比实时校验（所有系统之和提示是否超出/不足100%）
- [x] 4.4 占比数据写入系统条目，参与后续导出

---

## Phase 5：行为层描述字段

- [x] 5.1 在每个已选系统条目添加"核心循环描述"字段（行为层，一句话）
- [x] 5.2 明确 UI 区分：映射描述（定义层）与核心循环描述（行为层）的标签和位置
- [x] 5.3 对照现有映射描述内容，确认两者是否存在语义重叠，重叠时提示合并
- [x] 5.4 核心循环描述字段为可选，不填不阻断流程

---

## Phase 6：全局汇总视图

- [x] 6.1 基于已填写数据自动生成全局视图（系统名 + 占比 + 核心循环一览）
- [x] 6.2 全局视图为只读，不可在此处编辑数据
- [x] 6.3 全局视图不嵌入文档主体，作为独立面板或独立页存在
- [x] 6.4 全局视图支持按占比排序展示

---

## Phase 7：导出模块

- [x] 7.1 确认主文档导出结构（系统列表 + 映射描述 + 核心循环描述 + 占比）
- [x] 7.2 实现主文档导出（格式 TBD：markdown / json / docx）
- [x] 7.3 全局视图作为独立附页，导出时可选是否附带
- [x] 7.4 自定义系统与预设系统在导出中格式统一，来源标注 custom 标记

---

## Phase 8：流程收尾与校验

- [x] 8.1 整体流程节点校验：必填项检查（至少选一个系统、占比不为空）
- [x] 8.2 空状态处理：未选任何系统时的引导提示
- [x] 8.3 数据持久化：中途退出后草稿可恢复
- [x] 8.4 端到端流程走通测试（选项 → 自定义 → 访谈 → 占比 → 描述 → 全局视图 → 导出）

---

**总计：32 个原子任务**
**建议交付顺序：Phase 1 → 2 → 4 → 5 → 3 → 6 → 7 → 8**
（占比和描述字段优先于访谈模块，访谈是兜底而非主路径）
