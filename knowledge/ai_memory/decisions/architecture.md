## 2026-06-19：GUI 重构为主框架 + 标签切换

**决策**：将 `CommercialDesignApp(tk.Tk)` 改为 `CommercialDesignApp(tk.Frame)`，嵌入新的 `MainWindow(tk.Tk)` 主框架。顶部加标签栏切换"设计工作台" / "开发流水线"，不弹独立子窗口。

**理由**：
- 用户明确要求"打开时全屏"、"参考 VSCode 界面逻辑"
- 标签切换比独立 Toplevel 窗口更符合 IDE 风格
- 避免多窗口焦点管理问题
- 设计工作台和流水线共享底部面板（日志 + AI 对话），切换时上下文同步

**否决方案**：
- ❌ 独立 Toplevel 子窗口：会失去焦点，用户体验差，与 VSCode 风格不符
- ❌ 直接在 CommercialDesignApp 内加标签：代码耦合严重，难以维护

**影响范围**：
- 需修改 `core/ui/app_window.py`（CommercialDesignApp 改为 tk.Frame）
- 需新增 `core/ui/main_window.py`（主框架）
- 需修改 `core/ui/gui_app.py`（入口切换到 MainWindow）

## 2026-06-19：记忆系统放在 knowledge/ai_memory/

**决策**：AI 记忆系统存放在 `knowledge/ai_memory/` 目录，跟随 git 版本管理，不放在项目外的个人目录。

**理由**：
- 项目级知识应该共享，团队/多 AI 实例看到同样的记忆
- `knowledge/` 目录本身就是知识层，AI 记忆是知识的一种
- 遵守 CLAUDE.md 规则：不在根目录新建文件夹
- 与 Claude Code 的个人记忆系统（`~/.claude/projects/.../memory/`）互补而不重叠

**否决方案**：
- ❌ 放在根目录 `memory/`：违反 CLAUDE.md 规则
- ❌ 放在 `.claude/`：不跟随 git，无法共享
- ❌ 放在 `ucos/`：ucos 是游戏设计会话的 episodic 记忆，职责不同

**影响范围**：
- `knowledge/ai_memory/` 纳入 git 版本管理
- CLAUDE.md 顶部引用 `knowledge/ai_memory/INDEX.md`

## 2026-06-19：Unity 配置未完成时弹窗提示而非禁用按钮

**决策**：步骤03及以上运行时，若 Unity 路径未配置，点击"运行"后弹出 `messagebox.showwarning()`，精确提示缺失字段，不允许执行。而不是提前禁用"运行"按钮。

**理由**：
- 用户明确要求"不允许运行，弹出提示，哪个没配置就提示哪个"
- 弹窗能给出详细错误信息（development_path 缺失 vs editor_path 缺失）
- 禁用按钮会让用户不知道为什么不能点
- 引导用户点击"Unity配置"完成设置

**否决方案**：
- ❌ 直接禁用按钮：用户不知道原因
- ❌ 显示警告标记但允许继续：会导致步骤执行失败，浪费时间

## 2026-06-19：底部面板日志不随步骤切换清空

**决策**：日志面板显示本次会话的全部输出，不随用户切换步骤而清空。

**理由**：
- 用户可能需要对比多个步骤的日志
- 全局日志符合 IDE 风格（VSCode 的输出面板也是全局的）
- 如果需要清空，可以加"清空日志"按钮，由用户主动触发

**否决方案**：
- ❌ 切换步骤时清空：丢失上下文，对比日志困难
