# AutoDesignMaker Concept Source

Generated by AutoDesignMaker D4 DevFlow handoff.

## Layer 1 项目愿景
Submitted / accepted
- 项目定位: AI-assisted design-to-development automation workspace
  目的：merge structured design authoring, DevFlow stage orchestration, artifact validation, and UCOS memory into one movable project.
  依赖：design_tool data domains, DevFlow orchestrator, plugin manifest
  解锁：核心循环, 技术层, 生产层
- 平台: Windows desktop application with Python CLI pipeline support
  目的：run the GUI design tool and command-line development stages from the same project root.
  依赖：Tkinter runtime, Python environment, project settings
  解锁：运行时流程
- 目标玩家: game designers, technical designers, and Unity developers
  目的：support repeated design decisions, validation, handoff, and implementation review.
  依赖：domain schema coverage, artifact reports
  解锁：内容对象, 资源类型
- 商业模式: internal production tool and reusable automation framework
  目的：prioritize correctness, traceability, migration safety, and local execution over marketing presentation.
  依赖：portable paths, deterministic exports
  解锁：影响分析

## Layer 2 核心体验
Submitted / accepted
- 核心循环: load design knowledge -> choose structured decisions -> export Concept package -> run DevFlow stages -> review artifacts
  目的：make each design-to-development handoff explicit and testable.
  依赖：data/design, source_artifacts, outputs/artifacts
  解锁：stage 00 intake, stage validation
- 压力来源: manual handoff drift, stale absolute paths, missing source packages, and unverifiable development outputs
  目的：turn migration risks into validation checks before actual Unity project mutation.
  依赖：path manager, preflight checks
  解锁：migration audit
- 奖励节奏: immediate stage summaries, validation reports, package manifests, and build artifacts
  目的：give operators fast feedback after every design or development stage.
  依赖：artifact indexes, save records
  解锁：release packaging
